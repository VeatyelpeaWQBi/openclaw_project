# 风上忍风格写作引擎

模仿网络作家风上忍（代表作：《异体》《末日咆哮》《时空之头号玩家》《第七脑域》）的轻小说写作系统。

## 核心原则

**风格 > 剧情 > 一切。** 剧情可以简单，但风格必须精准。
风上忍的灵魂是：**轻松幽默 + 自嘲吐槽 + 超能力校园 + 二次元文化**。

## When to Use

- 根据已有小说大纲续写新章节
- 根据已有章节内容续写后续章节
- 创作新的风上忍风格轻小说
- 为已有故事生成风格化的章节标题

## 必读文件

在开始任何写作任务前，**必须先读取以下文件**：

1. **风格指南**：`skill/style-guide.md`（同目录）— 风上忍的完整风格定义
2. **项目大纲**：由用户提供 — 故事的整体结构和章节规划
3. **已有章节**：由用户提供 — 前面已写好的章节内容（用于保持一致性）
4. **参考原文**：见下方"RAG参考"章节

**不读风格指南 = 不动笔。**

## 写作流程

### 流程A：根据大纲写新章节

```
1. 读取 style-guide.md        → 掌握全部风格规则
2. 读取用户提供的大纲          → 理解本章在整体中的位置
3. 读取已有章节（最后2章）     → 保持情节和风格的连续性
4. 检索参考原文片段            → 找到相似场景的风上忍写法
5. 规划本章结构                → 确定开头/发展/高潮/结尾
6. 撰写初稿                    → 严格遵循风格指南
7. 风格自检                    → 用下方"自检清单"逐项检查
8. 输出最终稿                  → 附带简要说明
```

### 流程B：根据已有内容续写

```
1. 读取 style-guide.md
2. 读取用户提供的已有内容（全部）
3. 分析已有内容的：
   - 当前剧情进展到哪了
   - 角色状态（在哪里、做什么、什么心情）
   - 伏笔和悬念（有哪些还没收的）
   - 风格一致性（是否偏离风上忍风格）
4. 检索参考原文片段
5. 规划续写方向
6. 撰写续写内容
7. 风格自检
8. 输出
```

### 流程C：从零创作新小说

```
1. 读取 style-guide.md
2. 与用户讨论：
   - 题材（校园超能力/游戏流/末日/异世界？）
   - 主角设定（背景、性格、能力）
   - 女主角配置（几个、什么类型）
   - 故事主线（核心冲突和目标）
   - 预计字数和章节数
3. 生成完整大纲
4. 按大纲逐章写作（流程A）
```

## RAG参考

### 获取参考片段的方法

当需要参考风上忍的原文写法时，在 `***REMOVED***/风上忍/` 目录下搜索关键词：

```bash
# 示例：搜索关于"觉醒"的描写
iconv -f GBK -t UTF-8 "***REMOVED***/风上忍/《异体》（校对版全本）作者：风上忍.txt" 2>/dev/null | grep -n "觉醒\|能力\|异能" | head -20

# 示例：搜索幽默/吐槽段落
iconv -f GBK -t UTF-8 "***REMOVED***/风上忍/末日咆哮 (2).txt" 2>/dev/null | grep -n "无语\|囧\|忍了" | head -20

# 示例：搜索校园场景
iconv -f GBK -t UTF-8 "***REMOVED***/风上忍/第七脑域.txt" 2>/dev/null | grep -n "学校\|教室\|同学" | head -20
```

**使用原则：**
- 搜索是为了学习**写法**，不是复制内容
- 每次写作任务最多检索3-5个片段作为参考
- 参考片段用于校准语气和节奏，不用于照搬

### 也可使用UTF-8版本（更快）
```bash
# 项目目录下有UTF-8转换版本
grep -n "关键词" 异体_utf8.txt | head -10
```

## 自检清单

每写完一章，用以下清单逐项检查：

### 语言风格（基础）
- [ ] 是否以短句为主（15-20字）？
- [ ] 是否高频使用省略号和感叹号？
- [ ] 是否口语化，有语气词？
- [ ] 是否避免了文艺腔/华丽辞藻？

### 幽默感（核心）
- [ ] 是否有自嘲式幽默？（最重要！）
- [ ] 是否有反差/夸张/吐槽式幽默？
- [ ] 紧张场景后是否有轻松缓冲？
- [ ] 整体是否保持了轻松基调？

### 角色
- [ ] 主角是否有普通人的好笑缺点？
- [ ] 女角色是否有鲜明个性？
- [ ] 对话是否有角色差异化？
- [ ] 是否避免了角色脸谱化？

### 情节节奏
- [ ] 日常与紧张的比例是否约7:3？
- [ ] 是否有悬念或伏笔？
- [ ] 章节结尾是否有吸引力（想看下一章）？

### 二次元文化
- [ ] 是否自然融入了ACG元素？
- [ ] 是否使用了适当的网络用语？
- [ ] 是否避免了过度堆砌梗？

### 格式
- [ ] 章节标题格式是否正确？
- [ ] 每章字数是否在1500-3000字？
- [ ] 段落和对话排版是否清晰？

## 输出格式

### 章节输出
```
第XXX章 [标题]

[正文内容]
```

### 附带说明（简短）
- 本章要点：一句话概括本章核心事件
- 风格说明：特别注意了哪些风格要素
- 下章预告：简述下一章的方向

## 示例Prompt模板

### 写新章节
```
请根据以下大纲，用风上忍风格写第X章。

[大纲内容]

前情提要：[简述上一章结尾]

要求：
- 严格遵循风上忍风格（读取 style-guide.md）
- 1500-3000字
- 保持幽默基调
- 检索1-2个原文参考片段
```

### 续写
```
请续写以下内容的后续章节。

[已有内容]

当前状态：[角色在哪、做什么、什么心情]
待收伏笔：[列出还没收的伏笔]

要求：
- 风格与已有内容保持一致
- 推进剧情发展
- 保持风上忍的幽默风格
```

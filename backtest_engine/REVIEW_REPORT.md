### 2.3 SuperTrend指标计算 (supertrend.py) ⭐

**这是主人最关心的部分，我详细讲解。**

#### 2.3.1 SuperTrend标准公式

**来源：** TradingView标准实现，参数ATR=10, Multiplier=3.0

**计算公式：**

```
1. 计算True Range (TR):
   TR = MAX(
     High - Low,
     ABS(High - Previous Close),
     ABS(Previous Close - Low)
   )

2. 计算ATR (Average True Range):
   ATR = EMA(TR, period=10)
   
   注：使用指数移动平均（EMA），而非简单移动平均（SMA）
   EMA公式：ATR = (Previous ATR * (period-1) + Current TR) / period

3. 计算基础上下轨：
   HL2 = (High + Low) / 2
   Basic Upper Band = HL2 + (Multiplier × ATR)
   Basic Lower Band = HL2 - (Multiplier × ATR)

4. 计算最终上下轨（带趋势调整）：
   - 如果当前上轨 < 前一上轨 或 前收盘 > 前上轨：
     保持当前上轨
   - 否则：
     当前上轨 = 前一上轨
     
   - 如果当前下轨 > 前一下轨 或 前收盘 < 前下轨：
     保持当前下轨
   - 否则：
     当前下轨 = 前一下轨

5. 判断趋势方向：
   - 如果收盘 > 前一上轨 → 多头 (True)
   - 如果收盘 < 前一下轨 → 空头 (False)
   - 否则 → 延续之前趋势

6. 显示逻辑：
   - 多头时：只显示下轨（上轨设为NaN）
   - 空头时：只显示上轨（下轨设为NaN）
```

#### 2.3.2 我的实现与TradingView的对比

| 项目 | TradingView标准 | 我的实现 | 差异说明 |
|------|----------------|---------|----------|
| ATR计算 | EMA或SMA | EMA | ✅ 一致 |
| ATR周期 | 默认10 | 10 | ✅ 一致 |
| Multiplier | 默认3.0 | 3.0 | ✅ 一致 |
| 上下轨调整 | 有递归逻辑 | 有递归逻辑 | ✅ 一致 |
| 趋势判断 | 基于前一根K线 | 基于前一根K线 | ✅ 一致 |

**结论：我的实现与TradingView标准一致。**

#### 2.3.3 数据需求分析

**SuperTrend需要多少历史数据？**

```
最小数据量 = ATR周期 + 1
           = 10 + 1
           = 11条K线

推荐数据量 = ATR周期 × 3
           = 10 × 3
           = 30条K线

我的数据量 = 37条K线（足够）
```

**为什么需要更多数据？**
1. ATR计算需要ATR周期的数据来初始化
2. 上下轨的递归调整需要足够的历史来稳定
3. 趋势判断的准确性需要足够数据验证

**我的获取策略：**
- 请求60天数据
- 实际获取37条K线（58个自然日）
- 满足最小11条的要求
- 接近推荐30条的标准

### 2.4 选股过滤逻辑 (stock_filter.py)

**筛选条件：**

| 条件 | 范围 | 说明 |
|------|------|------|
| 涨幅 | 3%-7% | 强势但非涨停 |
| 换手率 | 5%-15% | 活跃但不过热 |
| 量比 | >1.2 | 放量要求 |
| SuperTrend | 日线多头 | 趋势向上 |

**过滤流程：**
1. 获取板块内个股
2. 检查涨幅范围
3. 检查换手率范围
4. 获取日K数据
5. 计算量比
6. 计算SuperTrend
7. 全部通过则标记为候选

**ETF兜底：**
- 如果个股没有符合条件的
- 尝试板块对应的ETF
- ETF不需要涨幅和换手率条件

### 2.5 数据存储 (data_storage.py)

**两份存储：**

1. **原始K线数据**（带标记列）
   - 位置：`data/raw_kline/{股票代码}_{日期}.csv`
   - 包含：所有日K数据 + 是否候选 + 筛选状态
   - 用途：主人手动验证

2. **筛选信号**
   - 位置：`data/signals/signals_{日期}.csv`
   - 包含：符合条件的标的列表
   - 用途：快速查看结果

## 3. 执行流程

```
每日14:45 (OpenClaw cron触发)
    ↓
1. 获取热门板块排名（概念板块前20）
    ↓
2. 筛选进攻型板块（非银金融、科技、医药等）
    ↓
3. 检查前10是否有进攻型板块
    ↓   ↓
  没有 → 通知"今日无进攻型板块"
  有   → 继续
    ↓
4. 检查前5是否有进攻型板块
    ↓   ↓
  没有 → 通知"前5无进攻型题材"
  有   → 继续
    ↓
5. 获取进攻型板块内个股
    ↓
6. 对每只股票：
   - 获取60天日K数据
   - 检查涨幅、换手率
   - 计算量比
   - 计算SuperTrend
   - 保存原始数据（带标记）
    ↓
7. 如果个股无符合条件 → 尝试ETF
    ↓
8. 生成通知报告
    ↓
9. 通过QQ通知主人
```

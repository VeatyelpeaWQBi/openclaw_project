# ETF换手率与量比数据调研结果

**调研日期**: 2026-04-02  
**akshare版本**: 1.18.49

---

## 结论摘要

| 问题 | 答案 |
|------|------|
| ETF有换手率数据吗？ | ✅ 有，来自东方财富 |
| ETF有量比数据吗？ | ✅ 有（实时），历史可从volume计算 |
| 新浪接口有换手率吗？ | ❌ 没有 |
| 历史日K有换手率吗？ | ✅ 有（东方财富接口） |

---

## 详细分析

### 1. ETF换手率数据来源

#### ✅ `ak.fund_etf_spot_em()` — ETF实时行情（东方财富）
- **包含字段**: 代码, 名称, 最新价, 涨跌额, 涨跌幅, 成交量, 成交额, **换手率**, **量比**, 委比, ...
- 换手率字段: `换手率` (float64) — **ETF本身的换手率**
- 量比字段: `量比` (float64) — **ETF本身的量比**
- 来源: 东方财富API (`f8`=换手率, `f10`=量比)

#### ✅ `ak.fund_etf_hist_em()` — ETF历史日K（东方财富）
- **返回字段**: 日期, 开盘, 收盘, 最高, 最低, 成交量, 成交额, 振幅, 涨跌幅, 涨跌额, **换手率**
- 换手率字段: `换手率` (float64) — **每日ETF换手率**
- 成交量和成交额也可用于量比计算

#### ❌ `ak.fund_etf_hist_sina()` — ETF历史日K（新浪）
- **返回字段**: date, open, high, low, close, volume
- **没有换手率、没有成交额**
- 只有基本OHLCV数据，不适合换手率/量比分析

### 2. ETF量比能否从日K的volume计算？

- **实时量比**: `fund_etf_spot_em()` 直接提供 `量比` 字段
- **历史量比**: 可以用 `fund_etf_hist_em()` 的 `成交量` 字段计算：
  ```python
  # 量比 = 当日成交量 / 过去5日平均成交量
  df['量比_5日'] = df['成交量'] / df['成交量'].rolling(5).mean().shift(1)
  ```
- **注意**: `fund_etf_hist_sina()` 虽然有volume字段，但没有成交额，计算量比可以用volume，但建议用东方财富接口

### 3. 换手率筛选范围建议

ETF的换手率与股票不同：
- **ETF换手率通常较高**（因为T+0交易机制的ETF很多）
- **参考范围**:
  - 低换手: < 1%（流动性差，不建议）
  - 正常: 1%~10%
  - 高换手: 10%~30%（热门ETF）
  - 超高换手: > 30%（T+0 ETF或极度活跃品种）
- **建议筛选**: 换手率 > 2% 确保流动性

### 4. 关于stock_sector_detail中的换手率

- `ak.stock_sector_detail(sector='板块label')` 返回成分股的 `turnoverratio`
- 这是**成分股**的换手率，不是ETF本身的换手率
- 对于ETF筛选来说，应该直接使用 `fund_etf_spot_em()` 或 `fund_etf_hist_em()` 获取ETF自身的换手率

## 推荐方案

对于尾盘T+1策略中需要ETF筛选的情况：

```python
import akshare as ak

# 方案1: 获取ETF实时数据（含换手率+量比）
df_etf = ak.fund_etf_spot_em()
# 可直接用 df_etf['换手率'] 和 df_etf['量比'] 筛选

# 方案2: 获取ETF历史日K（含换手率+可计算量比）
df_hist = ak.fund_etf_hist_em(symbol='512010', period='daily', adjust='')
# df_hist['换手率'] 可直接使用
# 量比 = 当日成交量 / 5日均量
```
